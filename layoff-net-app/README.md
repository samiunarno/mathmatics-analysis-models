# Layoff‑NET Full MERN‑TS Application

## Overview
This repository contains a **complete MERN stack** implementation for the Layoff‑NET research project:
- **Backend** – Express.js written in **TypeScript** (`/server`).  It exposes a `/predict` endpoint that loads the trained `layoff_model.pkl` (generated by the analysis pipeline) and returns predictions, probabilities, and evaluation metrics.
- **Frontend** – A modern **React** UI built with **Vite** and **TypeScript** (`/client`).  It visualises the dataset, model performance (ROC, confusion matrix, feature importance), and allows the user to upload a CSV of new companies to obtain live layoff‑risk predictions.
- **ML utilities** – The Python scripts (`final_analysis.py`, `ml_api.py`) produce the model artifact and supporting figures.

#### `src/routes/predict.ts`
```ts
import { Router, Request, Response } from "express";
import { spawn } from "child_process";
import path from "path";

export const predictRouter = Router();

// The Python helper that loads the model and returns JSON.
const PY_SCRIPT = process.env.PYTHON_SCRIPT || "../ml_api.py"; // relative to server root

predictRouter.post("/predict", async (req: Request, res: Response) => {
  const input = req.body; // expect {features: [...]}
  const py = spawn("python3", [PY_SCRIPT, JSON.stringify(input)], { cwd: path.resolve(__dirname, "..") });
  let stdout = "";
  let stderr = "";
  py.stdout.on("data", (data) => (stdout += data.toString()));
  py.stderr.on("data", (data) => (stderr += data.toString()));
  py.on("close", (code) => {
    if (code !== 0) {
      console.error("Python error:", stderr);
      return res.status(500).json({ error: "Model inference failed" });
    }
    try {
      const result = JSON.parse(stdout);
      return res.json(result);
    } catch (e) {
      console.error("JSON parse error", e);
      return res.status(500).json({ error: "Invalid response from model" });
    }
  });
});
```

#### Example **`src/components/UploadForm.tsx`**
```tsx
import { useState } from "react";
import axios from "axios";
import { Box, Button, Typography, Alert } from "@mui/material";

export default function UploadForm() {
  const [file, setFile] = useState<File | null>(null);
  const [result, setResult] = useState<any>(null);
  const [error, setError] = useState<string>("");

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!file) return setError("Please select a CSV file");
    const formData = new FormData();
    formData.append("file", file);
    try {
      const resp = await axios.post("http://localhost:5000/api/predict", formData, {
        headers: { "Content-Type": "multipart/form-data" },
      });
      setResult(resp.data);
      setError("");
    } catch (err) {
      setError("Server error – check console for details");
    }
  };

  return (
    <Box sx={{ p: 4 }}>
      <Typography variant="h5" gutterBottom>
        Upload new company data (CSV) for layoff‑risk prediction
      </Typography>
      <form onSubmit={handleSubmit}>
        <Button variant="contained" component="label">
          Choose CSV
          <input hidden type="file" accept=".csv" onChange={(e) => setFile(e.target.files?.[0] ?? null)} />
        </Button>
        <Button type="submit" variant="outlined" sx={{ ml: 2 }}>
          Predict
        </Button>
      </form>
      {error && <Alert severity="error" sx={{ mt: 2 }}>{error}</Alert>}
      {result && (
        <Box sx={{ mt: 3 }}>
          <Typography variant="h6">Prediction Results</Typography>
          <pre>{JSON.stringify(result, null, 2)}</pre>
        </Box>
      )}
    </Box>
  );
}
```

## 3️⃣ Running the whole stack together

### 3.1 Install **concurrently** (optional but convenient)
```bash
npm i -D concurrently
```
Add a **root‑level** `package.json` (outside `server` and `client`) with a single script:
```json
{
  "name": "layoff-net-app",
  "private": true,
  "scripts": {
    "dev": "concurrently \"npm run dev --prefix server\" \"npm run dev --prefix client\"",
    "build": "npm run build --prefix client && npm run build --prefix server"
  }
}
```
Now you can start everything with one command:
```bash
npm run dev   # launches API (http://localhost:5000) and UI (http://localhost:5173)
```
Press `Ctrl+C` to stop both processes.

### 3.2 Production build (optional)
1. **Backend** – compile TypeScript and start the compiled server:
   ```bash
   cd server
   npm run build   # produces ./dist
   npm start       # runs node dist/index.js
   ```
2. **Frontend** – create a static bundle:
   ```bash
   cd client
   npm run build   # outputs to ./dist
   ```
3. Serve the static React bundle with **Express** (add the following to `src/index.ts` after the API routes):
   ```ts
   import path from "path";
   app.use(express.static(path.join(__dirname, "../client/dist")));
   app.get("*", (req, res) => {
     res.sendFile(path.join(__dirname, "../client/dist/index.html"));
   });
   ```
   Re‑run `npm run build` in the server to include the new static‑serve code.

## 5️⃣ Helpful notes
- **CORS** – the backend already enables CORS for all origins. If you host the UI on a different domain, update the `cors` options accordingly.
- **Environment variables** – edit `/server/.env` to change the port or point to a different Python script.
- **Python side** – the script `ml_api.py` expects a JSON string on stdin and prints a JSON response. It loads the model (`layoff_model.pkl`) generated by the analysis pipeline. Ensure the `.pkl` file lives in the same directory as `ml_api.py`.
- **Figures** – copy the PNG files generated by the analysis (`fig1_eda_overview.png`, `fig3_roc_cm.png`, `fig4_architecture.png`, `fig5_code_snapshot.png`, `fig6_full_evaluation.png`, `fig7_metrics_summary.png`, `fig8_importance_learning.png`) into `client/src/assets/` so the UI can display them.
- **TypeScript strictness** – the provided snippets compile with `strict: true`. If you encounter type errors, you can relax the compiler options, but keeping strict mode helps catch bugs early.
